# PicoClaw: Ultra-Lightweight AI Agent Framework

PicoClaw is an ultra-efficient AI assistant framework built in Go, designed to run on minimal hardware (<10MB RAM, $10 devices) while maintaining full agent capabilities. It's a production-ready alternative to heavier frameworks when resource constraints matter.

## Core Characteristics

- **Ultra-Lightweight**: <10MB memory footprint (99% smaller than OpenClaw)
- **Minimal Cost**: Runs on $10 hardware (98% cheaper than Mac mini)
- **Lightning Fast**: 1-second boot time, even on 0.6GHz single-core processors
- **True Portability**: Single binary across RISC-V, ARM, x86 architectures
- **AI-Bootstrapped**: 95% of core code generated by AI agents

## Architecture Overview

PicoClaw follows a simplified agent architecture optimized for resource efficiency:

```
┌─────────────────────────────────────────┐
│         Channel Adapters                │
│  (Telegram, Discord, Slack, CLI)        │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│         Message Router                   │
│  (Session management, routing)           │
└──────────────┬──────────────────────────┘
               │
               ▼

               │
               ▼
┌─────────────────────────────────────────┐

└─────────────────────────────────────────┘
```

## Workspace Layout

PicoClaw stores all data in a configured workspace (default: `~/.picoclaw/workspace`):

```
~/.picoclaw/workspace/
├── sessions/          # Conversation sessions and history
├── memory/            # Long-term memory (MEMORY.md)
├── state/             # Persistent state (last channel, etc.)
├── cron/              # Scheduled jobs database
├── skills/            # Custom skills
├── AGENTS.md          # Agent behavior guide
├── HEARTBEAT.md       # Periodic task prompts (checked every 30 min)
├── IDENTITY.md        # Agent identity
├── SOUL.md            # Agent soul/personality
├── TOOLS.md           # Tool descriptions
└── USER.md            # User preferences
```

## Security Sandbox

PicoClaw runs in a sandboxed environment by default to prevent unauthorized access.

### Workspace Restriction

```json
{
  "agents": {
    "defaults": {
      "workspace": "~/.picoclaw/workspace",
      "restrict_to_workspace": true
    }
  }
}
```

When `restrict_to_workspace: true`, these tools are sandboxed:
- `read_file` - Can only read files within workspace
- `write_file` - Can only write files within workspace
- `list_dir` - Can only list workspace directories
- `edit_file` - Can only edit workspace files
- `append_file` - Can only append to workspace files
- `exec` - Can only execute commands within workspace context

### Command Blocking

Even with `restrict_to_workspace: false`, dangerous commands are blocked:

```go
var BLOCKED_PATTERNS = []string{
    "rm -rf",           // Bulk deletion
    "del /f",           // Windows deletion
    "rmdir /s",         // Directory removal
    "format",           // Disk formatting
    "mkfs",             // File system creation
    "diskpart",         // Disk partitioning
    "dd if=",           // Disk imaging
    "/dev/sd[a-z]",     // Direct disk writes
    "shutdown",         // System shutdown
    "reboot",           // System reboot
    "poweroff",         // Power off
    ":(){ :|:& };:",    // Fork bomb
}

func validateCommand(cmd string) error {
    for _, pattern := range BLOCKED_PATTERNS {
        if strings.Contains(cmd, pattern) {
            return fmt.Errorf("Command blocked: dangerous pattern detected")
        }
    }
    return nil
}
```

### Path Validation

```go
func validatePath(path string, workspace string) (string, error) {
    absPath, err := filepath.Abs(path)
    if err != nil {
        return "", err
    }
    
    absWorkspace, err := filepath.Abs(workspace)
    if err != nil {
        return "", err
    }
    
    if !strings.HasPrefix(absPath, absWorkspace) {
        return "", fmt.Errorf("Path outside workspace boundary")
    }
    
    return absPath, nil
}
```

## Heartbeat System (Cron-Based Tasks)

PicoClaw supports periodic autonomous tasks via the heartbeat system.

### Heartbeat Configuration

Edit `workspace/HEARTBEAT.md`:

```markdown
# Heartbeat Tasks

Check these tasks every 30 minutes:

1. **Email Monitoring**
   - Check for urgent emails
   - Summarize important messages
   - Alert on high-priority items

2. **Calendar Sync**
   - Review upcoming meetings (next 24 hours)
   - Identify scheduling conflicts
   - Send daily briefing at 8 AM

3. **System Health**
   - Check disk space
   - Monitor running processes
   - Alert if CPU > 80% for 5+ minutes
```

### Cron Job Management

```go
// Add a cron job
picoclaw cron add "0 8 * * *" "Send daily briefing"

// List all cron jobs
picoclaw cron list

// Remove a cron job
picoclaw cron remove <job_id>
```

### Programmatic Cron

```go
package main

import (
    "github.com/sipeed/picoclaw/pkg/agent"
    "github.com/robfig/cron/v3"
)

func main() {
    a := agent.New(agent.Config{
        Workspace: "~/.picoclaw/workspace",
    })
    
    c := cron.New()
    
    // Daily briefing at 8 AM
    c.AddFunc("0 8 * * *", func() {
        a.Execute("Generate daily briefing from calendar and emails")
    })
    
    // Check system health every hour
    c.AddFunc("0 * * * *", func() {
        a.Execute("Check system health and alert if issues found")
    })
    
    c.Start()
    a.Run()
}
```

## Multi-Channel Integration

### Telegram Bot

```go
package main

import (
    "log"
    tgbotapi "github.com/go-telegram-bot-api/telegram-bot-api/v5"
    "github.com/sipeed/picoclaw/pkg/agent"
)

func main() {
    bot, err := tgbotapi.NewBotAPI(os.Getenv("TELEGRAM_TOKEN"))
    if err != nil {
        log.Fatal(err)
    }
    
    a := agent.New(agent.Config{
        Workspace: "~/.picoclaw/workspace",
    })
    
    u := tgbotapi.NewUpdate(0)
    u.Timeout = 60
    updates := bot.GetUpdatesChan(u)
    
    for update := range updates {
        if update.Message == nil {
            continue
        }
        
        sessionID := fmt.Sprintf("telegram_%d", update.Message.Chat.ID)
        response := a.ProcessMessage(sessionID, update.Message.Text)
        
        msg := tgbotapi.NewMessage(update.Message.Chat.ID, response)
        bot.Send(msg)
    }
}
```

### Discord Bot

```go
package main

import (
    "github.com/bwmarrin/discordgo"
    "github.com/sipeed/picoclaw/pkg/agent"
)

func main() {
    dg, err := discordgo.New("Bot " + os.Getenv("DISCORD_TOKEN"))
    if err != nil {
        log.Fatal(err)
    }
    
    a := agent.New(agent.Config{
        Workspace: "~/.picoclaw/workspace",
    })
    
    dg.AddHandler(func(s *discordgo.Session, m *discordgo.MessageCreate) {
        if m.Author.ID == s.State.User.ID {
            return
        }
        
        sessionID := fmt.Sprintf("discord_%s", m.ChannelID)
        response := a.ProcessMessage(sessionID, m.Content)
        
        s.ChannelMessageSend(m.ChannelID, response)
    })
    
    dg.Open()
    select {}
}
```

```dockerfile

WORKDIR /app
COPY . .
RUN go build -o picoclaw cmd/picoclaw/main.go

FROM alpine:latest
RUN apk --no-cache add ca-certificates

WORKDIR /root/
COPY --from=builder /app/picoclaw .
COPY config.json .

CMD ["./picoclaw", "agent", "--config", "config.json"]
```

```yaml

services:
  picoclaw:
    build: .
    environment:
      - OPENAI_API_KEY=${OPENAI_API_KEY}
```

```go

├── sessions/     # Only recent sessions
├── memory/       # Essential memory only
```
