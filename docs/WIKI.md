# Crucible: The Modern Industrial Forge

**Version:** 1.0.0-rc.1  
**Aesthetic:** Modern Industrial Forge / Neobrutalism / Sci-Fi  
**Domain:** `forge-agents.space`  
**Deploy Target:** Vercel (Frontend), Supabase (Backend/Vector Storage)

---

## 1. Executive Summary

Crucible is a completely decentralized, multi-agent intelligence ecosystem
designed to simulate an autonomous industrial forge. Unlike generic Chatbots or
"assistants", Crucible employs specialized, named "Agent Personas" that
collaborate on complex tasks, from deep market signals tracking to
edge-deployment of micro-services.

---

## 2. Platform Architecture

### The Swarm Reactor (The Core)

The Swarm Reactor is the central hub where the six core AI nodes orchestrate
their tasks. Rendered via React Three Fiber (WebGL) and advanced SVGs, it
actively tracks the health, computing load, and token overhead of the entire
operation.

### The Agents

Crucible shuns standard roles ("Product Manager", "Developer") in favor of
industrial cybernetic personas:

- **Tungsten** (Core Architect): Structures foundational data schemas and
  topological graphs.
- **Cobalt** (Threat Vanguard): Scans edge perimeters for zero-day
  vulnerabilities and injection attacks.
- **Plasma** (Growth Injector): Accelerates market feedback loops and validates
  viral vectors.
- **Carbon** (Data Synthesizer): Bonds disparate market signals into unified
  JSON structures.
- **Titanium** (Deployment Frame): Hardens edge containers and prepares the
  Forge output for delivery.
- **Ignis** (Execution Engine): Ignites the build sequence and executes CI/CD
  pipelines.

---

## 3. Demand Radar: The Subsurface Scanner

Traditional startups use standard funnels or Kanban boards. Crucible utilizes
the **Seismic Demand Scanner**.

This specialized interface tracks market demand as "anomalies", drilling into
them for validation, smelting the core logic, and finally tracking the "Forged"
products.

- **Tracking:** High-resonance anomalies detected across API streams.
- **Drilling:** Deep analysis and proof-of-concept extraction via vector
  embeddings.
- **Smelting:** Intensive code-refinement within the autonomous loop.
- **Forged:** Final micro-products shipped automatically to the Vercel edge
  network.

---

## 4. Technical Stack

- **Frontend UI:** Next.js (App Router), React 18, TailwindCSS.
- **3D Visuals & Animation:** React Three Fiber, Drei, Framer Motion.
- **Backend Memory:** Supabase `pgvector` for autonomous context embedding,
  stored in the `market_research` table.
- **Routing:** Autonomous `next/server` API routes that scrape, summarize, and
  embed knowledge daily.

## 5. UI Philosophy: "Industrial Neobrutalism"

The interface rejects the clean, corporate "Glassmorphism" of modern SaaS
platforms. Instead, it utilizes heavy contrast, terminal-inspired typography
(`monospace`), glowing neon accents (Forge Orange `#ff8c00`, Matrix Green
`#00ff88`), visible grid lines, and "brick/steel" aesthetics. It looks not like
a software tool, but the dashboard of a heavy-machinery command center.
